# Phase 4 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Scope note (read first). This is the first phase that renders a net-new visual surface in
> v1.1.** Phase 3 shipped invisible infrastructure (`TourProvider` + `useTour()`); Phase 4 makes a
> coachmark actually appear on screen. The surface is a single component family:
>
> 1. `CoachmarkCard` — a fully custom React component (the `tooltipComponent` passed to react-joyride
>    v3.1). Joyride owns positioning + spotlight; the card owns ALL visual UI (CONTEXT D-02).
> 2. `CoachmarkRenderer` — a `"use client"` wrapper (loaded via `next/dynamic({ ssr: false })`, D-03)
>    that reads `useTour()`, drives joyride's `stepIndex`, and calls `advance()` / `dismiss()`.
> 3. A click-through cutout **overlay** (dim layer, `pointer-events: none`) above the z-40/z-50 nav
>    stack and z-[100] InstallGate (TOUR-02).
> 4. At least one `data-tour` anchor on the dashboard route to prove the engine end-to-end (TOUR-01).
>
> Teaching copy, embedded action surfaces, and cross-route sequencing are **Phase 5** — Phase 4 may use
> placeholder step copy to prove the card renders. This contract therefore freezes the **card visual
> shell + overlay + interaction behavior**, designed so Phase 5 can slot a content surface between the
> body text and the buttons with no structural change (D-05).

## Design System

Inherited verbatim from Phases 2–3. This phase adds **no new** primitive, token, or registry block. It
introduces ONE new bespoke component family (`CoachmarkCard` / `CoachmarkRenderer`) styled entirely
from existing tokens + utilities.

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (established, pre-existing — manual config, no `components.json`) |
| Preset | not applicable (theme in `tailwind.config.ts` + `src/app/globals.css` CSS variables) |
| Component library | Radix UI primitives (react-dialog, react-dropdown-menu) |
| Positioning engine | react-joyride v3.1 (npm dependency, NOT a shadcn registry block) — Floating UI under the hood (CONTEXT D-01) |
| Icon library | Lucide React 0.468.0 |
| Font | System default (Next.js / Tailwind base; no custom font face) |
| Theme name | "Monochrome Glass" — pure-black canvas, pure-white ink, translucent glass surfaces |
| Animation | `tailwindcss-animate` + existing keyframes in `globals.css` (`fade-up`, `state-in`), gated behind `prefers-reduced-motion` |

**Token source of truth:** `src/app/globals.css` `:root` CSS variables. The coachmark card is the
"brand dark card" (CONTEXT D-08) — but in this monochrome system the brand dark card IS a `.glass-card`
on the pure-black canvas (card surface = `--card` `0 0% 6%` / `bg-white/[0.045]` glass), NOT a separate
`bg-zinc-900`. Use the existing `.glass-card` utility or `bg-card` token; do **not** introduce a new
gray (`zinc-900`/`gray-900`) — that would break monochrome continuity. CONTEXT `<specifics>` named
`bg-zinc-900` as a candidate; this contract overrides it to the established `--card` token / `.glass-card`.

## Spacing Scale

8-point grid (established). The coachmark card uses these tokens only:

| Token | Value | Usage in CoachmarkCard / overlay |
|-------|-------|-------|
| xs | 4px | Dot-indicator gap (`gap-1`), inline icon gap |
| sm | 8px | Title-to-body gap (`space-y-2`), dot row top margin |
| md | 16px | Card inner padding (`p-4`), body-to-button gap |
| lg | 24px | Card inner padding on desktop if roomier (`p-6`); button-to-skip-link gap may stay at 12px |
| — | 12px (`mt-3`) | "Next →" button to "Skip tour" link gap (matches username-page seal/heading rhythm) |

**Card width:** 280–320px on desktop (CONTEXT discretion) → declared at **300px** (`w-[300px]`). On
mobile: full-width minus 16px margin each side (`mx-4` / `max-w-[calc(100vw-32px)]`).

**Exceptions:**
- **PWA safe-area insets (TOUR-03):** the overlay and any edge-anchored card must respect
  `env(safe-area-inset-top/right/bottom/left)`. When the card would land near a viewport edge, pad with
  `max(16px, env(safe-area-inset-*))` so it never sits under the notch, home indicator, or rounded
  corner on mobile standalone. This is the one place a non-grid value (the dynamic inset) is allowed.
- **Touch targets:** "Next →" button min height 44px (`h-11`) and "Skip tour" link min tap height 44px
  (padded hit area) for mobile accessibility, even though the link reads as small text.

## Typography

Card type scale (3 roles, 2 weights — within budget). Uses Tailwind base sizes, global line-height 1.5.

| Role | Size | Weight | Line Height | Where |
|------|------|--------|-------------|-------|
| Step title | 16px (`text-base`) | 600 (`font-semibold`) | 1.4 | `CoachmarkCard` heading (top of card) |
| Body text | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 (global) | 1–2 sentences of instructional copy (D-05) |
| Skip link | 13px (`text-[13px]`) / 14px (`text-sm`) | 400 | 1.5 | "Skip tour" secondary link (muted) |

Button label ("Next →") inherits the shadcn `Button` primitive type: 14px (`text-sm`) `font-semibold` —
do not override. Two weights total: 400 (body/link) + 600 (title/button). No third weight.

## Color

"Monochrome Glass" inherited as-is. The coachmark card is dark-on-black with white ink; the primary
action is solid white (the system's only accent), matching the established `Button` default variant.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` (`--background` `0 0% 0%`) | The dim overlay tint sits over the black canvas |
| Secondary (30%) | `#0F0F0F` card / white-low-alpha glass (`--card` `0 0% 6%` / `bg-white/[0.045]` `.glass-card`) | `CoachmarkCard` surface |
| Accent (10%) | `#FFFFFF` (`--primary` / `--accent` `0 0% 100%`) | **Reserved list below** |
| Destructive | `0 72% 51%` (`--destructive`) | **Not used in this phase** |

**Accent (solid white) reserved for — explicit list, never "all interactive elements":**
- The single primary **"Next →" button** (`Button` default variant: `bg-white text-black`) — one per card.
- The **current step's dot** in the progress indicator (filled solid white).
- The **spotlight ring edge** highlighting the anchored element (the cutout's inner border, if a
  border-radius ring is used — CONTEXT discretion).

**Supporting (non-accent) values:**
- **Step title + body text:** white (`text-foreground` / `text-white`).
- **"Skip tour" link:** muted gray, lower contrast (`text-white/60` ≈ `--muted-foreground` `0 0% 60%`)
  with `hover:text-white/85` — visually secondary so advance stays dominant (D-06, D-08).
- **Past-step dots:** secondary white (`bg-white/40`). **Future-step dots:** dim (`bg-white/15`).
- **Overlay dim layer:** `rgba(0,0,0,0.5)` — the recommended box-shadow cutout
  (`box-shadow: 0 0 0 9999px rgba(0,0,0,0.5)`) on the highlighted element (CONTEXT discretion). This
  layer is `pointer-events: none` so it is click-through (TOUR-02). The cyan `--c-action` (`#03d5ff`)
  glass-bevel internal is **NOT** used as a coachmark color — it stays inside `.glass-*` bevels only.

Error/validation color rule (inherited, preserved by absence): this phase renders no error UI in the
coachmark — joyride mount/positioning failure degrades silently (the tour simply does not appear, app
stays usable, consistent with Phase 3 D-06). Nothing is recolored to `--destructive`.

## Copywriting Contract

> **Phase 4 uses PLACEHOLDER step copy to prove the card renders.** Final teaching copy (gym, challenge,
> money, Shortcut — outcome-framed, consequence-first, "stakes not stats" per UX-04) is **Phase 5**
> (CONTEXT `<deferred>`). The contract below fixes the **structural / control copy** (buttons, skip,
> degraded behavior) that ships in Phase 4 and the **voice** any placeholder must honor.

| Element | Copy |
|---------|------|
| Primary CTA | **"Next →"** (default; CONTEXT discretion allows step-specific "Got it →" — must be ≤ 20 chars). Calls `advance(currentStepId)`. |
| Skip / dismiss affordance | **"Skip tour"** — secondary text link below the Next button (D-06). Calls `dismiss()`. Not a header X. |
| Placeholder step title (POC) | e.g. **"This is your dashboard"** — neutral, replaced in Phase 5. Lowercase-calm, no exclamation, no gamified praise. |
| Placeholder step body (POC) | 1–2 plain sentences proving wrapping/positioning, e.g. **"Coachmarks will spotlight each part of SweatPact, one at a time. Real lessons land in the next phase."** |
| Empty state | n/a — the coachmark only mounts when `useTour().isActive` is true (a step is pending). When inactive it renders nothing (no empty card, no placeholder shell). |
| Error state | n/a — joyride mount / unmounted-anchor / positioning failure = silent no-op (app fully usable). No error text, toast, or retry prompt rendered (mirrors Phase 3 D-06). |
| Destructive confirmation | **None.** "Skip tour" is non-destructive (resumable; per ONB-04 the user is never blocked or nagged) — no confirm dialog. |

**Voice guardrails (for the Phase-5 author, locked now):** consequence-first, "stakes not stats",
lowercase-calm, no gamified completion praise or badges (REQUIREMENTS Out-of-Scope). Brand-voiced
completion moment ("pact is live") is Phase 6 (UX-03) — not authored here.

## Interaction Contract — Coachmark Engine (the load-bearing deliverable)

This phase **consumes** the frozen Phase-3 `useTour()` API (`{ currentStepId, isActive, advance,
dismiss }`) and adds the rendering behavior below. It does **not** extend `useTour()` (`isComplete` /
`progress` remain deferred — Phase 3 D-07 / CONTEXT `<deferred>`).

| Aspect | Contract |
|--------|----------|
| Mount gate | `CoachmarkRenderer` renders the joyride overlay only when `useTour().isActive === true` (a step is pending, not dismissed). Otherwise renders `null`. |
| Step drive | `currentStepId` → joyride `stepIndex` (controlled). Advancing calls `advance(currentStepId)`; the re-derived `currentStepId` moves joyride to the next anchor. |
| Anchor readiness (TOUR-01) | A step's spotlight shows **only once its `data-tour` target is actually mounted**. If the anchor is absent/unmounted, do NOT spotlight empty space — skip/hold until mounted (joyride callback / `MutationObserver` on the anchor). |
| Click-through (TOUR-02) | Dim overlay is `pointer-events: none` (box-shadow cutout recommended). The page beneath stays fully interactive; input is never trapped. |
| Z-index (TOUR-02) | Overlay + card render above TopNav (z-40), MobileNav/Radix Dialog (z-50), and InstallGate (z-[100]) — i.e. **above z-[100]** (e.g. `z-[110]`+), via the dedicated `#tour-root` portal (CONTEXT integration points + CONCERNS.md). |
| Portal isolation | joyride `portalElement` → `document.getElementById("tour-root")`; `#tour-root` placed in the root or `(tabs)` layout at the same level as other portals (`<Toaster>`). Keeps joyride out of the Radix portal hierarchy. |
| Radix dialog coexistence (D-04, TOUR-02) | When any `[role="dialog"][data-state="open"]` is present, **hide/pause** the coachmark; re-show on close. Detect via `MutationObserver` on `document.body` watching Radix `data-state`. |
| Keyboard (TOUR-04) | **Enter / Space** = advance; **Escape** = dismiss. Focus is moved to the card's primary button when a step appears (`aria-live` announcement of title+body). Focus is **NOT trapped** — Tab cycles the card's own controls only when focus is explicitly there; the rest of the page stays reachable (CONTEXT discretion: no focus trap). |
| Reduced motion (TOUR-04) | Behind `window.matchMedia("(prefers-reduced-motion: reduce)")`: set joyride `floaterProps.disableAnimation` and disable the card's own enter/exit CSS transitions (mirrors `globals.css` reduced-motion block). |
| Safe-area (TOUR-03) | Card + overlay padding uses `max(16px, env(safe-area-inset-*))` so coachmarks position correctly within PWA safe-area insets on notched mobile standalone. |
| Scroll behavior | Try joyride `disableScrollParentFix: true` by default (App Router scroll-lock workaround, CONTEXT `<specifics>`). |
| Card structure (D-05) | Top→bottom: step **title** → 1–2 sentence **body** → **dot indicator** row (5 dots = `STEPS.length` from `src/lib/onboarding/steps.ts`) → primary **"Next →"** button → muted **"Skip tour"** link. A surface slot between body and buttons is reserved for Phase 5 (no markup added now). |
| Dot indicator (D-07) | One dot per `STEPS` entry (5). Current = solid white; past = `bg-white/40`; future = `bg-white/15`. No "2 of 5" text. |
| POC anchor (CONTEXT discretion) | Add at least one `data-tour="<step-id>"` attribute to an element **unconditionally mounted** on the dashboard route (e.g. `data-tour="challenge"` on the primary CTA or `data-tour="schedule"` on the weekly-goal card). Executor picks based on what renders without conditional/streamed gating, to satisfy TOUR-01 cleanly. |

## Acceptance Checklist (phase-specific, for executor + auditor)

The coachmark engine is visually + interaction-correct when ALL are true:

- [ ] A coachmark card appears on the dashboard route only when `useTour().isActive` is true, anchored to
      a real, mounted `data-tour` element — never spotlighting empty/streamed space (TOUR-01).
- [ ] The dim overlay is click-through (`pointer-events: none`); the page beneath stays interactive and
      input is never trapped (TOUR-02).
- [ ] The overlay/card render above InstallGate (z-[100]) and the nav stack, isolated in `#tour-root`,
      not conflicting with Radix portals (TOUR-02).
- [ ] Opening any Radix dialog hides the coachmark; closing it restores the coachmark (D-04).
- [ ] Coachmarks respect `env(safe-area-inset-*)` on mobile standalone — never under notch/home
      indicator/rounded corner (TOUR-03).
- [ ] Enter/Space advances, Escape dismisses, focus moves to the card on appear, `aria-live` announces
      the step, and focus is NOT trapped (TOUR-04).
- [ ] `prefers-reduced-motion` disables joyride + card animations (TOUR-04).
- [ ] Card matches the contract: white-ink dark `.glass-card`/`bg-card` surface (NOT a new gray), 16px
      semibold title, 14px body, 5-dot indicator (current=white), white "Next →" button, muted
      "Skip tour" link below it.
- [ ] No new color, type role, spacing token (beyond the safe-area dynamic inset), primitive, icon, or
      shadcn registry block is introduced. `--c-action` cyan is not used as a coachmark color.
- [ ] `useTour()` is consumed unchanged — no `isComplete`/`progress` added; only `currentStepId`,
      `isActive`, `advance`, `dismiss` are used.
- [ ] joyride mount / unmounted-anchor / positioning failure degrades silently — app fully usable, no
      error UI rendered.
