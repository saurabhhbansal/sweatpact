  <action>
    Modify `src/app/layout.tsx` to integrate the three PostHog components created in Plan 02.

    Required imports to add (use @/ import paths):
    - `import { Suspense } from "react"` — React built-in, no package
    - `import { PostHogProvider } from "@/components/posthog-provider"`
    - `import { PostHogPageview } from "@/components/posthog-pageview"`
    - `import { PostHogIdentity } from "@/components/posthog-identity"`

    Layout structure change:
    - Wrap the entire `<body>` element with `<PostHogProvider>...</PostHogProvider>`. The PostHogProvider must be a SIBLING of `<body>`, not inside it, because the body itself is not a client component — wrap the body as: `<PostHogProvider><body className="min-h-screen">...</body></PostHogProvider>`.

    Wait — the correct structure for Next.js App Router: the html element is the outermost, then body inside. PostHogProvider must wrap the body's CHILDREN (it is a client component and cannot wrap the html/body elements themselves in a way that conflicts with Next.js). The correct pattern is:

    `<html lang="en"><body className="min-h-screen"><PostHogProvider>...children...</PostHogProvider></body></html>`

    OR — and this is the pattern from the research — PostHogProvider wraps the body's content:

    Inside the `<body className="min-h-screen">`:
    - First child: `<PostHogProvider>` open tag
      - `<Suspense fallback={null}><PostHogPageview /></Suspense>` — MUST be inside Suspense; useSearchParams in PostHogPageview causes static generation bailout if not wrapped
      - `<PostHogIdentity />` — no Suspense needed (no useSearchParams)
      - `<SplashScreen />` — existing, unchanged
      - `<InstallGate>{children}</InstallGate>` — existing, unchanged
      - `<div id="tour-root" />` — existing, unchanged (comment preserved)
    - `</PostHogProvider>` close tag

    So the body children order is: PostHogProvider wraps Suspense+PostHogPageview, PostHogIdentity, SplashScreen, InstallGate+children, tour-root div.

    Preserve: all existing imports (InstallGate, SplashScreen), the metadata export, the viewport export, and the JSDoc comment on tour-root.
    Do NOT remove any existing content.
    Do NOT add any className or styling.

    After editing, verify visually that the JSX nesting is correct — PostHogProvider is a wrapper around ALL existing body children.
