<objective>
Set up the /ingest reverse proxy, exclude it from middleware, and add a PWA service worker bypass (FOUND-04).

Purpose: Without the reverse proxy, PostHog events are blocked by ad blockers and tracking-protection extensions. Without `skipTrailingSlashRedirect: true`, PostHog's trailing-slash endpoints (like `/e/`) get 404s. Without the middleware exclusion, the session-cookie refresh runs on every analytics request — unnecessary and potentially harmful. Without the SW fetch handler, installed PWA users lose all analytics.

Output: `next.config.mjs` with three ordered rewrites and skipTrailingSlashRedirect; `src/middleware.ts` matcher with `ingest` excluded; `public/sw.js` with a conditional fetch passthrough.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /ingest (Next.js rewrite) → us.i.posthog.com | Analytics events cross from app origin to PostHog cloud via server-side proxy |
| Service worker → /ingest fetch passthrough | SW intercepts fetch events; bypass sends /ingest requests directly to network |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-07 | Tampering | next.config.mjs rewrites destination | accept | Rewrite destinations are hardcoded to PostHog hostnames at build time; no runtime redirect to arbitrary host is possible |
| T-07-08 | Denial of Service | SW fetch listener catch-all | mitigate | Fetch bypass is conditional on /ingest/ prefix only; a bare catch-all would intercept navigation requests and break the app offline — enforced by the condition check |
| T-07-09 | Information Disclosure | /ingest proxy leaks internal network topology | accept | /ingest is a standard PostHog reverse-proxy pattern; the destination hostname is already public (PostHog docs); no internal-only services are exposed |
| T-07-10 | Tampering | middleware exclusion removes security checks from /ingest | accept | /ingest paths carry only analytics events (no auth, no financial data); skipping session-cookie refresh on these paths has no security impact |
</threat_model>
