# Phase 6 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase:** Skip-on-Complete, Replay & Completion Hardening (final phase of v1.1).
**Requirements covered:** PROG-03 (replay from Settings), UX-03 ("pact is live" completion moment). Skip-on-complete (PROG-02 hardening) and legacy cleanup are non-visual / removal-only.

---

## New / Changed Visual Surfaces

| Surface | Type | Where | Source decision |
|---------|------|-------|-----------------|
| **"Pact is live" overlay** | Full-screen Radix Dialog | Fires on `/groups` when first active challenge detected; shown once | UX-03, D-01/D-02/D-03 — the emotional peak of v1.1 |
| **Replay app tour button** | Ghost/secondary button row | Inside existing `SettingsForm`, in the iOS Shortcuts / setup region | PROG-03, D-05 |
| Skip-on-complete behavior | No new UI | `deriveCurrentStep()` fires with real probes before first coachmark paints | D-07, "Specifics" — must feel instant, no flash |
| Legacy wizard pages | **Removal only** | Delete `/onboarding/{gym,schedule,shortcut}` + `step-indicator.tsx` | D-08/D-09 — no new UI |

The "pact is live" overlay is the **primary new visual surface** and the only one requiring net-new visual design. Everything else reuses or removes existing UI.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual shadcn-style system; no `components.json`, CLI registry not used) |
| Preset | not applicable |
| Component library | Radix UI primitives (`@radix-ui/react-dialog`) wrapped in `src/components/ui/*` |
| Icon library | lucide-react 0.468.0 |
| Font | System font stack (Tailwind default; no custom `fontFamily` declared) |
| Theme | "Monochrome Glass" — dark-only, pure black canvas (`--background: 0 0% 0%`), white ink (`--foreground: 0 0% 100%`). All tokens are HSL CSS variables in `src/app/globals.css`. |
| Surface recipes | `.glass-card` (static cards on black), `.glass-liquid` (floating bars/dialogs with backdrop blur). Reuse — do not invent new surface classes. |

**Reuse mandate:** This phase introduces no new tokens, no new fonts, no new surface recipes. The "pact is live" overlay is built from existing tokens, the existing `Dialog` primitive, and existing `animate-*` utilities.

---

## Spacing Scale

Existing system uses Tailwind's default 4px-base scale. Declared values for this phase (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps (`gap-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `mt-2`) |
| md | 16px | Default element padding (`p-4`, `mt-4`) |
| lg | 24px | Overlay content vertical rhythm, section padding (`p-6`, `space-y-6`) |
| xl | 32px | Major gaps between overlay headline block and CTA |
| 2xl | 48px | Vertical breathing room above/below the overlay headline on tall viewports |

Exceptions:
- **44px / `h-11`** minimum tap target for the overlay CTA and replay button (mobile PWA; matches existing `CoachmarkCard` `h-11` controls and Button `lg` `h-12`).
- **`env(safe-area-inset-bottom)`** padding on the overlay CTA region (PWA standalone — same pattern as `ChangePasswordButton` sheet `paddingBottom: max(env(safe-area-inset-bottom), 1.25rem)`).

---

## Typography

Sizes drawn from the existing established scale (3 sizes + 1 display for the overlay headline). 2 weights only — normal (400) and semibold (600).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | normal (400) | 1.5 |
| Label / CTA | 14px (`text-sm`) | semibold (600) | 1.5 |
| Heading | 16–18px (`text-base`/`text-lg`) | semibold (600) | ~1.4 |
| Display (overlay headline only) | 28–32px (`text-3xl`, `text-4xl` on `sm:`) | semibold (600) | 1.2 |

The overlay headline is the one place a display size is justified — it carries the emotional weight of "Your pact is live." Do not exceed `text-4xl`; the brand is sharp, not shouty. Body line-height inherits the global `1.5` set on `html, body`.

---

## Color

The system is intentionally monochrome — white is the dominant ink and the accent. There is no chromatic accent color in normal UI; chroma is reserved for semantic state only.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` `hsl(0 0% 0%)` pure black | Overlay backdrop, page canvas |
| Secondary (30%) | `.glass-card` / `.glass-liquid` surfaces (`white/[0.045]`–`white/10` over black) | Overlay content panel, replay button surface |
| Accent (10%) | `--primary` / `--foreground` `hsl(0 0% 100%)` white | The single primary CTA fill ("Let's go →"), the headline ink |
| Destructive | `--destructive` `hsl(0 72% 51%)` red | NOT used in this phase — no destructive actions here |

**Accent reserved for:** the overlay's single white-filled primary CTA button ("Let's go →") and the white headline ink. Nothing else in this phase uses a filled white accent. The replay button is deliberately NOT accent — it is a quiet ghost/secondary control (see Components).

**Brand color note:** No confetti palette, no green "success" celebration. The "pact is live" moment is a *stakes* moment, not a *celebration* — it stays in the monochrome system. The red `--destructive` token must NOT be borrowed to imply danger; the consequence is communicated through copy, not color.

## Components

### "Pact is live" overlay

| Property | Contract |
|----------|----------|
| Primitive | Radix `Dialog` (`@radix-ui/react-dialog`) — gives focus trap, Escape, `aria` labelling. Use `Dialog.Root` + `Dialog.Portal` + `Dialog.Overlay` + `Dialog.Content`. |
| Layout | **Full-screen** (`fixed inset-0`), NOT the centered `max-w-lg` card from `src/components/ui/dialog.tsx`. Build a bespoke full-bleed `Dialog.Content` (or override `className`) following the `InstallGate` / `notifications-overlay` full-screen pattern. |
| Z-index | **`z-[120]`** — must render ABOVE `COACHMARK_Z_INDEX = 110` (the tour) and above InstallGate `z-[100]`. This is the topmost surface in the app. |
| Backdrop | Pure black or near-black with blur (`bg-black/90 backdrop-blur-xl`), consistent with `notifications-overlay`. No dimmed-through-to-content look — this is a takeover. |
| Entrance animation | `animate-overlay-in` for the backdrop + `animate-fade-up` or `animate-sheet-in` for the content block. Reuse existing keyframes; both auto-disable under `prefers-reduced-motion` (already wired in `globals.css`). No confetti, no bounce, no scale-pop celebration. |
| Headline | Display size (`text-3xl`/`sm:text-4xl`), semibold, white. |
| Body | `text-sm`/`text-base`, `text-white/70`. One short consequence-framed line. |
| CTA | **Single** primary button, white fill (`Button` default variant), full-width on mobile, `h-12`/`size="lg"`, label "Let's go →". Dismisses overlay, stays on `/groups`. |
| Dismiss affordance | The single CTA is the only dismiss path shown. Escape key still works (Radix). No `X` close button in the corner — the moment has one exit, forward. |
| Safe area | CTA region respects `env(safe-area-inset-bottom)` in PWA standalone. |
| Icon | Optional single lucide glyph above headline (e.g. a lock/flame motif) — monochrome white, `h-8`/`h-10`. Keep restrained; copy carries the moment. |
| Shown-once | Overlay suppressed after first view via persisted seen-flag (mechanism is Claude's discretion per D-03 — prefer a `completed_steps` entry e.g. `pact_live_seen` for cross-device persistence). UI never re-renders the overlay once seen.|

### Replay app tour button

| Property | Contract |
|----------|----------|
| Component | `Button` `variant="ghost"` or `variant="secondary"` — a quiet control, NOT the white accent fill. Ghost matches the muted "Skip tour" precedent in `CoachmarkCard`. |
| Placement | Inside existing `SettingsForm` (`src/app/(tabs)/settings/client.tsx`), in the onboarding-adjacent region — under the "iOS Shortcuts" section or as a row near the gym/schedule settings. Do NOT add a new top-level `SectionHeader` or restructure the page. |
| Layout option | May follow the existing settings-row pattern (`glass-card` row with label + `›` chevron) for visual consistency with surrounding rows, OR a plain ghost button — either is acceptable; match whichever reads cleaner in context. |
| Label | "Replay app tour" — not "Restart", not "Tutorial", not "Start over". |
| Sublabel (if row form) | Short, e.g. "Walk through the app again" — `text-xs text-white/50` per existing row sublabels. |
| Icon | Optional lucide `RotateCcw` or `Compass`, `h-[18px]` `text-white/60` to match `UserCircle`/`MapPin` row icons. |
| Behavior | Calls `PATCH /api/onboarding-progress` with the replay signal; reactivates the tour. No loading spinner needed beyond disabling during the in-flight PATCH. Auto-skip means already-done steps fast-forward silently. |
| Tap target | `h-11` minimum. |

### Skip-on-complete (no visual component)

| Property | Contract |
|----------|----------|
| Visual requirement | Auto-skipped steps must produce **zero flash** — the coachmark for an already-done step must never paint. `deriveCurrentStep()` must resolve the first non-done step using real probe data BEFORE the first coachmark renders (D-07, Specifics). |
| Acceptance | On replay or fresh start with gym+schedule+shortcut already done, the tour opens directly on the first incomplete teaching step with no intermediate frame showing a completed step's card. |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (overlay) | `Let's go →` |
| Overlay headline | `Your pact is live.` |
| Overlay body | `Real money's on the line now. Show up — or pay up.` |
| Replay button label | `Replay app tour` |
| Replay button sublabel (if row form) | `Walk through the app again` |
| Empty state | Not applicable — no new empty state this phase (existing `EmptyStatePactCTA` from Phase 5 unchanged). |
| Error state (replay PATCH fails) | Inline, quiet: `Couldn't restart the tour. Try again.` — `text-sm text-destructive` matching the existing `ChangePasswordButton` error pattern. Non-blocking; button re-enables. |
| Destructive confirmation | None — this phase has no destructive actions. |

**Voice rules (UX-04):** consequence-first, sharp, second-person. "Show up or pay up" energy. Banned: confetti language, "Congratulations!", "You did it!", "Woohoo", exclamation-stacking, generic celebration. The overlay states a fact about stakes; it does not congratulate. One exclamation mark maximum across the whole overlay (the headline period is preferred over an exclamation — `Your pact is live.` reads sharper than `Your pact is live!`).

## Interaction & Motion

| Interaction | Contract |
|-------------|----------|
| Overlay entrance | Backdrop fades (`animate-overlay-in`, ~0.2s ease-out), content rises/fades (`animate-fade-up`/`animate-sheet-in`, ~0.28s). One coordinated entrance, no staggered confetti. |
| Overlay exit | On CTA click or Escape, Radix `data-state="closed"` drives the existing faster exit (`overlay-out` 0.15s). User remains on `/groups` with their live challenge visible. |
| Reduced motion | All overlay animations auto-disable via the existing `@media (prefers-reduced-motion: reduce)` block (covers `.animate-overlay-in`, `.animate-fade-up`, `.animate-sheet-in`). No extra work — just use those utilities. |
| Replay button | `active:scale-[0.98]` from Button variants; disabled state at `opacity-50` during PATCH. |
| Skip-on-complete | Instant — the absence of motion is the contract. No flash, no transition between skipped steps. |

## Accessibility

| Concern | Contract |
|---------|----------|
| Overlay focus | Radix `Dialog` traps focus inside the overlay; focus lands on the single CTA on open. |
| Overlay labelling | `Dialog.Title` = headline, `Dialog.Description` = body (use the `DialogTitle`/`DialogDescription` primitives or `aria-labelledby`/`aria-describedby`). |
| Escape | Radix Escape closes the overlay (equivalent to the CTA). |
| Replay button | Real `<button>`, keyboard-operable, visible `focus-visible:ring-2 focus-visible:ring-ring/80` from Button base. |
| Contrast | White-on-black headline and white-fill CTA exceed WCAG AA on the black canvas; `text-white/70` body meets AA at the body size. |
| Tap targets | Overlay CTA `h-12`, replay control `h-11` — both ≥44px. |
