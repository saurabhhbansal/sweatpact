# Phase 5 — UI Design Contract

> Visual and interaction contract for Cross-Route Walkthrough & Teaching Content.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This is a brownfield phase.** Every token below is the project's *existing*
> "Monochrome Glass" system (defined in `src/app/globals.css` + `tailwind.config.ts`)
> and the *locked* Phase 4 coachmark card (`src/components/tour/coachmark-card.tsx`).
> Phase 5 adds NO new design primitives — it extends the Phase 4 card with an inline
> surface slot, adds anchors on 3 routes, and writes brand-voiced teaching copy.
> Do not invent new colors, fonts, or spacing values.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn-style components (Radix primitives + CVA `Button`), tokens via CSS variables |
| Preset | Monochrome Glass — custom, established at v1.0 (not a fresh shadcn init this phase) |
| Component library | Radix UI (`@radix-ui/react-dialog`, `react-dropdown-menu`) + local `@/components/ui/*` |

| Coachmark z-index | `COACHMARK_Z_INDEX = 110` (above InstallGate z-[100], nav z-40/z-50) — locked in Phase 4, do not change |

## Spacing Scale

Tailwind default 4px-based scale. Phase 5 uses these existing tokens (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`) | Progress-dot gaps |
| sm | 8px (`mt-2`, `p-2`) | Title→body, body→dots gaps inside the coachmark card |
| md | 12px (`mt-3`, `p-3`) | Card button stack gaps; checklist item padding |
| md+ | 16px (`p-4`, `mx-4`) | Coachmark card padding; card horizontal viewport margin |
| lg | 24px (`gap-6`, `space-y-6`) | Dashboard section spacing (checklist ↔ daily strip) |
| xl | 32px | Empty-state CTA vertical breathing room |

Touch targets: all interactive controls (Next, Skip tour, surface Continue/Skip, checklist links, empty-state CTA, Practice check-in) are **min 44px tall** (`h-11`) — carried from Phase 4 D-06 / TOUR-04.

Card sizing: width `w-[300px] max-w-[calc(100vw-32px)]` (Phase 4 locked). When a surface is embedded (D-01/D-02), the card may widen to a bounded max and the **surface slot** is `max-h-[calc(80vh-8rem)] overflow-y-auto` so the card never exceeds `~80vh` and respects PWA safe-area insets (TOUR-03).

Exceptions: none — surface slot max-height (80vh) is a viewport constraint, not a new spacing token.

## Typography

Carried verbatim from the locked Phase 4 coachmark card. Body line-height 1.5 is the global default (`globals.css` line 52). Exactly 2 weights, 3 sizes in the tour surface.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Card title / step heading | 16px (`text-base`) | semibold (600) | 1.2 (heading) |
| Body / instructional copy | 14px (`text-sm`) | normal (400) | 1.5 |
| Skip link / micro-label | 13px (`text-[13px]`) | normal (400) | 1.5 |
| Checklist item label | 14px (`text-sm`) | normal (400) → completed items may use white/60 strike-free muted | 1.5 |

Weights: **normal (400)** + **semibold (600)** only. No light, no bold, no black.
Display tier: not used in this phase (no hero/marketing headings in the tour surface).

## Color

The system is intentionally monochrome — value (white opacity) carries hierarchy, not hue. "Accent" here is **pure white** used as the primary-action fill, which is the system's single deliberate emphasis.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 0%)` black `--background` | Page canvas, overlay scrim base |
| Secondary (30%) | `.glass-card` (white @ 4.5% + bevel) / `--card: 0 0% 6%` | Coachmark card, checklist container, embedded surface bg |
| Accent (10%) | `hsl(0 0% 100%)` white `--primary` | See reserved-for list below |
| Destructive | `hsl(0 72% 51%)` `--destructive` | Not used in Phase 5 (no destructive flows) |
| Success (status) | `hsl(142 71% 45%)` `--success` | Checklist item *completed* checkmark only; practice-checkin confirmation pulse |
| Muted text | `white/60` (`--muted-foreground`) | Skip link, completed/secondary copy, future progress dots |

**Accent (white) reserved for:** the primary `Button` fill (`bg-white text-black`) on the card's Next / surface Continue / empty-state "Start your first pact" CTA; the **current** progress dot (`bg-white`); the spotlight cutout ring. NOT used for body text emphasis, NOT for every interactive element (skip links and secondary actions stay `white/60–85`).

Progress dots (locked Phase 4): current = `bg-white`, past = `bg-white/40`, future = `bg-white/15`.

## Copywriting Contract

Voice: **consequence-first, "stakes not stats," brand-voiced, outcome-framed** (UX-04). Never gamified, never "Get started" generic, never badge/reward language (Out-of-Scope). Plain, direct, a little blunt.

### Per-step teaching copy (5 steps)

| Step (id) | Title | Body |
|-----------|-------|------|
| schedule (`schedule`) | "Set your weekly goal" | "How many days a week are you showing up? This is the bar you and your partner are held to." |
| gym (`gym`) | "Pick your gym" | "We verify check-ins by location. Set the gym you actually train at — no gym, no proof." |
| challenge — self-starter (`challenge`) | "Start a pact" | "Challenge your gym partner with real money on the line. Skip a day you owe — show up and you don't." |
| challenge — invited (`challenge`, pendingCount>0) | "Your partner challenged you" | "This is where you respond. Accept the pact to put real money on the line — or decline." |
| money (`money`) | "This is the scoreboard that matters" | "Not streaks — money. What you've earned, what you owe, and how it settles every week." |
| shortcut (`shortcut_viewed`) | "Check in from your phone" | "iOS users: one tap via the Shortcut. Everyone else: manual check-in works the same. Try a practice run below — it won't count." |

### Standard contract elements

| Element | Copy |
|---------|------|
| Primary CTA (card, surface steps) | Surface's own "Continue" (advances on real action) — **no standalone "Next →" on surface steps** (D-03). Teaching-only steps (challenge, money) keep "Next →". |
| Practice check-in button | "Practice check-in" (clearly labeled, D-05). After ~400ms success pulse → `advance('shortcut_viewed')`. Sub-label/helper: "Practice only — never counts toward stakes." |
| Empty state heading (dashboard, no challenges) | "No stakes yet" |
| Empty state body + CTA (UX-02) | "Your partner is waiting. Put real money on the line." → button: **"Start your first pact"** |
| Getting-started checklist heading (UX-01) | "Getting started" |
| Checklist items (4) | "Set your gym" · "Start a pact" · "See the money model" · "Set up check-ins" (each shows a `success` checkmark when its TEACHING_KEY is in `completedSteps`; whole checklist hides once all 4 done) |
| Skip control (every step) | "Skip tour" — muted `white/60` text link below the card body/surface (ONB-04, locked Phase 4) |
| Error state | Surfaces own their own error states (Phase 2, unchanged). Tour adds none. If a surface fails to save, its existing inline error shows; the step does NOT auto-advance. Practice check-in cannot error (no API call, D-05). |
| Destructive confirmation | None — Phase 5 has no destructive actions. Decline-invite uses the existing notifications `respond()` flow, not a tour-owned destructive control. |

## Motion

Reuse existing animation utilities only (`globals.css`). All must no-op under `prefers-reduced-motion` (the global `@media` block already covers `animate-*`).

| Interaction | Animation | Duration |
|-------------|-----------|----------|
| Card appears / step change | `.animate-fade-up` (existing, locked Phase 4) | 0.28s |
| Practice check-in success | Brief pulse/checkmark (tailwindcss-animate), reduced-motion → instant | ≤400ms |
| Checklist item completes | `.animate-state-in` (existing) on the check toggle | 0.24s |

No new keyframes. No motion on route transitions beyond the renderer's existing anchor-gate reveal.
