  <action>
Create src/lib/analytics/server.ts as a new file in the existing analytics module directory.

Imports: import PostHog from posthog-node; import type EventName from @/lib/analytics/events.

Export a single async function captureServerEvent with signature: captureServerEvent(distinctId: string, event: EventName, properties?: Record string unknown): Promise void.

The function body is wrapped in a try/catch where the catch block is completely empty (swallows all errors silently — analytics must never interrupt business logic).

Inside the try block:
1. Read apiKey from process.env.NEXT_PUBLIC_POSTHOG_KEY. If apiKey is falsy, return immediately (no-op — covers test/CI environments where the key is not set).
2. Hardcode host as the string "https://eu.i.posthog.com" — this is the direct EU PostHog ingestion endpoint confirmed from next.config.mjs rewrites. Do NOT use NEXT_PUBLIC_POSTHOG_HOST which is /ingest (a browser reverse proxy URL, invalid for server-to-server calls).
3. Create a new PostHog instance (per-call, not a module singleton): new PostHog(apiKey, { host, flushAt: 1, flushInterval: 0 }). The flushAt:1 and flushInterval:0 config forces immediate single-event delivery rather than batching.
4. Call client.capture({ distinctId, event, properties }).
5. Call await client.shutdown() to flush synchronously before the function returns.

No other exports. No module-level state.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01-01 | Information Disclosure | captureServerEvent properties parameter | mitigate | Callers must pass only Supabase UUIDs and enum/constant values as properties — never email, name, location coordinates, or any PII. Document in function JSDoc comment: "properties must contain only UUIDs and enum values." |
| T-08-01-02 | Information Disclosure | NEXT_PUBLIC_POSTHOG_KEY in server code | accept | NEXT_PUBLIC_POSTHOG_KEY is a public write-only ingestion key already exposed in the browser SDK. Using it server-side is standard PostHog practice and creates no new secret surface. |
| T-08-01-03 | Denial of Service | captureServerEvent error swallowing | accept | Silent catch is intentional: analytics failures must never impact business logic availability. The tradeoff is missing analytics data on PostHog SDK failures, which is acceptable. |
