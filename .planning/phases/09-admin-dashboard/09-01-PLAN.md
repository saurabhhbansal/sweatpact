<objective>
Build the fail-closed owner gate that protects every `/admin` surface (ADMIN-01). `parseAdminUserIds()` turns the comma-separated `ADMIN_USER_IDS` env var into a trimmed allow-list; `requireOwner()` revalidates the session via `getUser()` and calls Next.js `notFound()` for anyone not on the list.

Purpose: This is the single most security-critical control in Phase 9 — it is what stops a non-owner from seeing cross-user financial and analytics data.
Output: `src/lib/admin-auth.ts` (two exports) + co-located unit tests. No UI, no data fetching.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser session cookie → RSC | An attacker may present a forged/expired Supabase cookie to reach `/admin` |
| env config → authorization | A misconfigured/empty `ADMIN_USER_IDS` must never widen access |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Spoofing | requireOwner() session check | mitigate | Use `supabase.auth.getUser()` (server round-trip revalidation), never `getSession()` (cookie-only, spoofable) |
| T-09-02 | Information Disclosure | /admin route existence | mitigate | `notFound()` (404) for all deny paths — non-owner cannot distinguish absent vs forbidden |
| T-09-03 | Elevation of Privilege | empty ADMIN_USER_IDS | mitigate | `parseAdminUserIds` returns `[]` and `requireOwner` calls `notFound()` when list is empty — fail closed |
| T-09-04 | Information Disclosure | server-only secret scope | mitigate | `import "server-only"` at top of `admin-auth.ts`; never expose to client bundle |
</threat_model>
