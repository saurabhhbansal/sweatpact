# Phase 9 — UI Design Contract

> Visual and interaction contract for the owner-gated `/admin` dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Source of truth:** This phase inherits SweatPact's established "Monochrome Glass" design system (`src/app/globals.css`, `tailwind.config.ts`, `src/components/ui/*`). The admin shell is a *standalone* layout (no tab nav, no TourProvider) but uses the **same brand tokens**. Do NOT invent new tokens — reuse the existing CSS variables.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/Radix — manually vendored (no `components.json`). Do NOT run `shadcn init`; the system is established and CLAUDE.md requires new work to match it. |
| Preset | not applicable (hand-maintained token set in `globals.css`) |
| Component library | Radix UI primitives wrapped in `src/components/ui/` (Card, Button, Badge, Dialog, etc.) |
| Icon library | Lucide React (`lucide-react` 0.468.0) |
| Font | System sans-serif stack (Tailwind default — no custom webfont loaded) |
| Chart library | `recharts` (new dependency this phase — DASH-02 line chart). DASH-05/06 use CSS percentage bars, not recharts. |
| Theme | "Monochrome Glass" — pure-black canvas, white ink, glass surfaces. Dark-only (no light mode). |

**Reusable component inventory (already exists — do not rebuild):**
`Card` / `CardHeader` / `CardTitle` / `CardContent` (`rounded-[2rem] glass-card`), `Button` (+ `buttonVariants`), `Badge` (variants: default, secondary, destructive, success, warning, muted), `Dialog`, `DropdownMenu`, `Input`, `Label`. Use `formatCents()` from `@/lib/money` for all financial display.

## Spacing Scale

Declared values (all multiples of 4 — matches existing usage `gap-3`, `p-4`, `p-5`, `gap-2`):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1` / `mt-1` | Icon gaps, inline label-to-value spacing |
| sm | 8px | `gap-2` / `p-2` | Compact element spacing, badge padding |
| md | 12px | `gap-3` | Default gap between panels in the dashboard grid |
| lg | 16px | `p-4` / `gap-4` | Inner panel padding (data-dense cards), grid column gap |
| xl | 20px | `p-5` | Standard `CardHeader` / `CardContent` padding (matches `ui/card.tsx`) |
| 2xl | 24px | `py-6` | Section breaks between Supabase block and PostHog block |
| 3xl | 32px | `pt-8` | Page top padding below the fixed header |

**Layout grid:** desktop `grid-cols-2 gap-3` (or `gap-4`), mobile `grid-cols-1 gap-3`. Page container `max-w-5xl mx-auto px-4` (wider than the app's `max-w-md` — admin is a desktop-first data surface).

**Exceptions:**
- Touch targets keep existing button heights: `h-10` (40px) default, `h-9` (36px) sm, `h-12` (48px) lg. Date-range segmented control pills must stay ≥ 36px tall for touch.
- Card border radius stays `rounded-[2rem]` (the established glass-card radius) — do not reduce for admin.

## Typography

System sans stack. Sizes/weights mirror established usage in `src/app/(tabs)/dashboard/page.tsx` and `ui/card.tsx`.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display (metric) | 28px (`text-3xl`) | 700 bold | 1.1 (`leading-tight`) | KPI figures — total revenue, user count, settlement totals |
| Heading (panel title) | 16px (`text-base`) | 600 semibold | 1.0 (`leading-none`) | `CardTitle` for each of the 6 panels |
| Body | 14px (`text-sm`) | 400 regular | 1.5 | Descriptions, row text, empty/error copy |
| Label / eyebrow | 12px (`text-xs`) | 500 medium | 1.4 | Metric labels, axis labels, "This range" eyebrows. Uppercase eyebrows use `tracking-[0.08em]`–`[0.18em]` + `text-white/45`–`/55` (established pattern). |

**Weight contract:** primary pair is **400 regular + 600 semibold**. Two documented exceptions inherited from the existing system, allowed only as specified:
- **700 bold** — reserved exclusively for Display metric figures (large numbers).
- **500 medium** — reserved for small-caps eyebrow labels and `Badge` text.

No other weights. No font sizes between or outside the four roles above.

## Color

All values are existing `globals.css` tokens — reference via `hsl(var(--token))` or the Tailwind color aliases, never hardcoded hex.

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `--background` `hsl(0 0% 0%)` → `#000000` | Page canvas behind everything |
| Secondary (30%) | `glass-card` surface `rgba(255,255,255,0.045)` over black (token `--card` `hsl(0 0% 6%)`) | All 6 panel surfaces, header bar, tooltip/popover backgrounds |
| Accent (10%) | `--primary` / `--accent` `hsl(0 0% 100%)` → `#FFFFFF` | See reserved list below |
| Destructive | `--destructive` `hsl(0 72% 51%)` → `#DC2626` | Negative financial deltas, geo-fail chart series, error-state text |
| Success (semantic) | `--success` `hsl(142 71% 45%)` | Positive financial figures, verified check-in series, healthy/goal-met indicators |
| Warning (semantic) | `--warning` `hsl(38 92% 50%)` | At-risk / pending metrics, funnel drop-off warnings |

**Accent (white) reserved for — explicit list, nothing else:**
- Primary Display metric figures (the headline number in each KPI card)
- The primary/most-important data series line in the DASH-02 chart (total check-ins)
- The active pill in the date-range segmented control (`bg-white text-black`, matching `Button` default)
- The "Back to app" link on hover/focus
- Focus rings (`ring-ring/80`, existing `--ring` = white)

Accent is NOT for: panel borders, body text, every interactive element, or decorative emphasis. Secondary text uses `text-white/55`–`/70`; hairlines use `border-white/10`–`/18` (token `--border`).

**Chart palette (recharts — DASH-02 multi-series line chart):**
| Series | Color | Token |
|--------|-------|-------|
| Total check-ins (primary) | white | `hsl(0 0% 100%)` |
| Verified | emerald | `hsl(var(--success))` |
| Geo-fail | red | `hsl(var(--destructive))` |
| Manual vs Shortcut (secondary split) | muted white | `hsl(0 0% 60%)` (`--muted-foreground`) |
| Grid lines / axes | `hsl(0 0% 18%)` (`--border`) at low opacity |
| Tooltip surface | `glass-card` background + `border-white/10` |

DASH-05/06 CSS percentage bars: filled portion `bg-white` (or `bg-success` for positive ratios), track `bg-white/10`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Back to app" — single right-aligned link in the fixed header (admin is read-only; this is the only navigational action) |
| Date-range control | Segmented control with three options: "7 days" · "30 days" · "90 days" (default 30 days). Drives `?range=7d\|30d\|90d`. |
| Header title | "SweatPact Admin" (left-aligned, `text-base font-semibold`) |
| Empty state heading | "No data yet" |
| Empty state body | "Events started 2026-06-28. Check back once members generate activity." (per CONTEXT — honest, sets expectation for < 50-user product) |
| Error state (PostHog) | "Analytics unavailable — couldn't reach PostHog. Showing the last cached snapshot; refresh to retry." |
| Error state (Supabase) | "Couldn't load financial data. Refresh the page to retry." |
| Error state (whole page) | "Couldn't load the admin dashboard. Refresh, and check that your session is still valid." |
| Destructive confirmation | None — the admin dashboard is strictly read-only. No mutations, no destructive actions in this phase. |

**Tone:** factual and operator-facing (the audience is the SweatPact owner, not an end user). No marketing voice, no exclamation marks. Numbers over adjectives.

## Panel Layout Contract

6 panels, single scrollable page, source-grouped (CONTEXT decision):

| # | Panel | Source | Visual |
|---|-------|--------|--------|
| DASH-01 | Financial overview | Supabase (admin client) | KPI cards — `formatCents()`, success/destructive deltas |
| DASH-02 | Check-in trend | Supabase | recharts `LineChart`, multi-series, date-range driven, ISO-week buckets |
| DASH-03 | User overview | Supabase | KPI cards / compact table |
| DASH-04 | Onboarding funnel | PostHog | Stepped bars, drop-off in `warning`/`destructive` |
| DASH-05 | Feature adoption | PostHog | Metric cards + CSS % bars |
| DASH-06 | Engagement & retention | PostHog | Metric cards + CSS % bars |

Order: financial/Supabase block first (top), PostHog block below (`border-t border-white/10` or a `24px` gap as the divider). Grid: `grid-cols-2` desktop, `grid-cols-1` mobile. Each panel is a `Card` (`rounded-[2rem] glass-card`). Each PostHog panel renders the empty state independently (data may arrive per-panel).
