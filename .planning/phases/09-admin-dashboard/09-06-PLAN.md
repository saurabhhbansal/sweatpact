<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /admin route | Authorization must happen before any cross-user data is fetched |
| service-role client → cross-user data | RLS-bypassing admin client must only run inside the owner-gated route |
| `?range` searchParam → queries | User input must be whitelisted before influencing a query window |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-14 | Elevation of Privilege | createAdminClient in page | mitigate | Layout `await requireOwner()` runs before the page renders; the RLS-bypass client is reachable only after the 404 gate passes |
| T-09-15 | Tampering | `?range` searchParam | mitigate | `z.enum(["7d","30d","90d"]).catch("30d")` whitelists range; mapped to fixed integer days via `rangeToDays` — never interpolated raw into a query |
| T-09-16 | Information Disclosure | error rendering | mitigate | try/catch renders fixed copy; Next redirect/notFound signals re-thrown; no stack trace leaked to client (Security V7) |
| T-09-17 | Spoofing | gate placement | mitigate | Authorization in the RSC layout (not middleware) per RESEARCH anti-pattern; `getUser()` revalidation in `requireOwner` |
</threat_model>
