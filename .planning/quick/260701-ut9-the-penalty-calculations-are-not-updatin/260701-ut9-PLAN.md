<objective>
Fix: weekly missed-goal penalties are not updating after the end of a week.

Root cause: `runEnforcement` (src/lib/enforcement.ts) only runs the weekly goal check when the previous local day is a Sunday (`dayOfWeekFor(day) === 0`). That is a single-shot, non-idempotent-at-the-week-level trigger with no catch-up. The daily Vercel cron (`0 19 * * *`) scans every profile sequentially and reconciles just one day per run. If the one Sunday→Monday run is skipped, times out, or lands during a deploy, that week's penalties are never created and there is no mechanism to recover them — so a user's owed amount silently fails to update after the week ends.

Fix: run the weekly reconciliation for the *most recently closed* ISO week on *every* daily run, reusing the existing idempotent, self-reversing `reconcileUserWeek`. Monday-run behavior stays identical to today; every other day of the following week gains a safe catch-up pass so a single missed run is healed automatically.

Purpose: restore the core product guarantee — skipping a week must produce the correct debt — under real-world cron flakiness.
Output: a new exported `reconcileMostRecentClosedWeek` helper, its test, and a rewired `runEnforcement`.
</objective>
